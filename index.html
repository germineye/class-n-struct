BÀI 01

Abstraction và Class

Thay vì quản lý từng đối tượng bằng một đống biến và hàm rời rạc, ta định nghĩa một kiểu dữ liệu chung rồi tạo ra bao nhiêu đối tượng tùy ý.

1. Vì sao cần abstraction?

Giả sử game có 100 con quái. Mỗi con đều có máu và có thể nhận sát thương. Viết riêng 100 biến và 100 hàm là cách tự hành xác. Ta gom đặc điểm chung đó thành một kiểu Monster.

StateTrạng thái hiện tại của object, ví dụ Health.
BehaviorHành vi object có thể thực hiện, ví dụ TakeDamage().

User-defined type là kiểu dữ liệu do chính ta tạo để mô tả đối tượng trong chương trình.

2. Khai báo class và tạo object

cpp
class Monster {
public:
    int Health{150};

    void TakeDamage(int Damage) {
        Health -= Damage;
    }
};

int main() {
    Monster Bonker;
    Monster Basher;
}
  • Monster là class.
  • Bonker và Basher là object, cũng có thể gọi là instance.
  • Quá trình tạo object từ class gọi là instantiation.
  • Dấu chấm phẩy sau dấu } của class là bắt buộc.

3. Class member và toán tử .

Biến và hàm nằm bên trong class được gọi chung là class member. Biến là data member; hàm là member function hoặc method.

cpp
#include <iostream>

int main() {
    Monster Bonker;

    std::cout << Bonker.Health << '\n';
    Bonker.TakeDamage(25);
    std::cout << Bonker.Health << '\n';
}

Toán tử . là member access operator. Mỗi object có bản sao riêng của data member, nên sửa Bonker.Health không ảnh hưởng đến Basher.Health.

4. Tách phần khai báo và định nghĩa hàm

Member function có thể được khai báo trong class rồi định nghĩa bên ngoài bằng scope resolution operator ::.

cpp
class Monster {
public:
    void TakeDamage(int Damage);

private:
    int Health{150};
};

void Monster::TakeDamage(int Damage) {
    Health -= Damage;
}
Tóm lại

Class là bản mô tả chung; object là thực thể cụ thể. Class gom state và behavior vào cùng một kiểu dữ liệu.

BÀI 02

Encapsulation và Access Specifier

Không để code bên ngoài sửa thẳng dữ liệu nội bộ. Mọi thay đổi nên đi qua một lối vào có kiểm soát.

1. Class invariant và contract

Class invariant là quy tắc object luôn phải giữ, chẳng hạn Health không bao giờ âm. Lời đảm bảo đó cũng có thể gọi là contract.

cpp
void TakeDamage(int Damage) {
    Health -= Damage;

    if (Health < 0) {
        Health = 0;
    }
}

Hàm trên giữ được invariant, nhưng vô nghĩa nếu bên ngoài vẫn có thể viết Goblin.Health = -999;. Vì vậy dữ liệu cần được ẩn.

2. public và private

cpp
class Monster {
public:
    void TakeDamage(int Damage) {
        Health -= Damage;

        if (Health < 0) {
            Health = 0;
        }
    }

private:
    int Health{150};
};
  • public: code bên ngoài object được phép truy cập.
  • private: chỉ member function của chính class được truy cập.
  • Trong class, member mặc định là private.

3. Getter và setter

Getter cho phép đọc dữ liệu private. Setter cho phép thay đổi dữ liệu nhưng vẫn áp dụng kiểm tra.

cpp
class Monster {
public:
    int GetHealth() const {
        return Health;
    }

    void SetHealth(int NewHealth) {
        Health = NewHealth < 0 ? 0 : NewHealth;
    }

private:
    int Health{150};
};
Đừng tạo getter/setter theo phản xạ.

Chỉ công khai thao tác mà code bên ngoài thực sự cần. Nếu field nào cũng có setter vô điều kiện thì private gần như mất tác dụng.

4. API của class

API của class gồm tên class và các public member. Phần private là implementation detail: có thể thay đổi mà không buộc code sử dụng class phải sửa theo.

Tóm lại

Dữ liệu private, hành vi public. Class tự chịu trách nhiệm giữ object ở trạng thái hợp lệ.

BÀI 03

Constructor và Destructor

Constructor bảo đảm object vừa được tạo ra đã có dữ liệu hợp lệ. Destructor chạy tự động khi object không còn tồn tại.

1. Constructor giải quyết vấn đề gì?

Nếu không có constructor, ta có thể tạo object rồi quên gán dữ liệu cần thiết. Constructor gom việc tạo object và thiết lập dữ liệu ban đầu thành một thao tác duy nhất.

Không có constructor

Monster Goblin;
Goblin.SetName("Bonker");
Goblin.SetHealth(150);

Có constructor

Monster Goblin{
    "Bonker", 150
};

Ở cách thứ hai, không thể có một Monster bị thiếu tên hoặc thiếu máu nếu constructor yêu cầu cả hai giá trị.

2. Viết constructor đầu tiên

cpp
#include <string>

class Monster {
public:
    Monster(std::string InitialName, int InitialHealth)
        : Name{InitialName}, Health{InitialHealth} {}

private:
    std::string Name;
    int Health;
};

int main() {
    Monster Goblin{"Bonker", 150};
}
  • Monster(...) là constructor vì nó trùng tên class.
  • Constructor không có return type, kể cả void.
  • "Bonker" đi vào parameter InitialName.
  • 150 đi vào parameter InitialHealth.
  • Sau dấu :, hai data member được khởi tạo từ hai parameter đó.
Khi chạy dòng Monster Goblin{"Bonker", 150};, C++ tạo object và tự gọi constructor phù hợp. Ta không gọi constructor như một function thông thường.

3. Member initializer list là gì?

Phần nằm giữa dấu : và thân constructor {} gọi là member initializer list.

cpp
Monster(std::string InitialName, int InitialHealth)
    : Name{InitialName}, Health{InitialHealth} {
    // Tới đây, Name và Health đã được khởi tạo xong.
}

Đọc đoạn này từ trái sang phải:

  • Name{InitialName}: tạo data member Name bằng giá trị của parameter InitialName.
  • Health{InitialHealth}: tạo data member Health bằng giá trị của parameter InitialHealth.
  • Sau đó thân constructor mới chạy.
Thứ tự thật sự theo khai báo.

Data member luôn được khởi tạo theo thứ tự chúng xuất hiện trong class, không phải theo thứ tự viết trong initializer list. Vì vậy nên viết hai thứ tự giống nhau để tránh đọc nhầm.

4. Default constructor và nhiều cách tạo object

Default constructor là constructor cho phép tạo object mà không truyền argument.

cpp
#include <string>

class Monster {
public:
    Monster() = default;

    Monster(std::string InitialName, int InitialHealth = 150)
        : Name{InitialName}, Health{InitialHealth} {}

private:
    std::string Name{"Unknown"};
    int Health{150};
};

int main() {
    Monster UnknownMonster;          // Dùng Monster()
    Monster Goblin{"Bonker"};        // Health mặc định là 150
    Monster Boss{"Basher", 500};     // Health là 500
}
  • = default yêu cầu compiler tạo phần triển khai mặc định.
  • InitialHealth = 150 là default argument.
  • Nhiều constructor cùng tên nhưng khác parameter gọi là overloading.
  • Khi đã tự khai báo một constructor, compiler không còn tự cấp default constructor. Muốn tạo object không argument, phải khai báo lại như ví dụ.

5. Constructor bị ambiguous

Mỗi cách gọi phải khớp chính xác một constructor. Nếu compiler thấy nhiều lựa chọn ngang nhau, lời gọi bị ambiguous.

cpp
class Monster {
public:
    Monster(int Level) {}
    Monster(int Health) {} // Lỗi: chữ Level và Health không tạo type mới
};

Cả hai constructor đều có chữ ký Monster(int). Tên parameter không được dùng để phân biệt overload; compiler chỉ quan tâm type và số lượng parameter.

6. Destructor chạy lúc nào?

Destructor có tên class kèm dấu ~. Nó không có return type và không nhận parameter.

cpp
#include <iostream>

class Session {
public:
    Session() {
        std::cout << "Bat dau session\n";
    }

    ~Session() {
        std::cout << "Ket thuc session\n";
    }
};

void Run() {
    Session CurrentSession;
    std::cout << "Dang chay\n";
} // CurrentSession bị hủy tại đây

int main() {
    Run();
    std::cout << "Da quay lai main\n";
}

Thứ tự output:

output
Bat dau session
Dang chay
Ket thuc session
Da quay lai main

Với class chỉ dùng các kiểu tự quản lý như std::string và std::vector, thường không cần tự viết destructor.

Tóm lại

Constructor chạy khi object được tạo; initializer list khởi tạo data member; destructor chạy khi vòng đời object kết thúc.

BÀI 04

Struct và Aggregate Initialization

Struct phù hợp với một nhóm dữ liệu nhỏ cần được giữ và truyền đi cùng nhau.

1. Tại sao không dùng ba biến rời?

Ba biến rời

float PlayerX{1.0F};
float PlayerY{2.0F};
float PlayerZ{3.0F};

Một object Vector3

Vector3 PlayerPosition{
    1.0F, 2.0F, 3.0F
};

x, y và z cùng mô tả một vị trí, nên gom chúng thành một type giúp truyền, trả về và lưu trữ dễ hơn.

2. Tạo và sử dụng struct

cpp
struct Vector3 {
    float x;
    float y;
    float z;
};

int main() {
    Vector3 Position;

    Position.x = 1.0F;
    Position.y = 2.0F;
    Position.z = 3.0F;
}
  • Vector3 là tên type mới.
  • Position là một object của type Vector3.
  • Member của struct mặc định là public, nên truy cập trực tiếp bằng dấu ..

3. Struct khác class ở đâu?

structclass
Member mặc địnhpublicprivate
Inheritance mặc địnhpublicprivate
Function, constructor, destructorCóCó
Cách dùng thường gặpDữ liệu đơn giảnKiểu có invariant và hành vi

Về khả năng ngôn ngữ, struct và class gần như giống nhau. Khác biệt lớn nhất trong code thường là ý đồ thiết kế:

  • Dùng struct khi các field có thể được đọc và sửa trực tiếp mà không làm object mất hợp lệ.
  • Dùng class khi cần che giấu dữ liệu và kiểm soát cách object thay đổi.

4. Aggregate initialization

Với một aggregate đơn giản, ta có thể truyền giá trị cho các member ngay trong dấu {} mà không cần tự viết constructor.

cpp
struct Vector3 {
    float x;
    float y;
    float z;
};

int main() {
    Vector3 Position{1.9F, 2.6F, 0.3F};
}

C++ ghép giá trị theo thứ tự khai báo:

1.9Fđược gán cho x
2.6Fđược gán cho y
0.3Fđược gán cho z
Không ghép theo tên.

Nếu đổi thứ tự khai báo member, ý nghĩa của cùng một danh sách giá trị cũng đổi theo. Với type có nhiều field cùng type, constructor hoặc designated initializer có thể dễ đọc hơn.

5. Giá trị mặc định cho struct

cpp
struct Vector3 {
    float x{0.0F};
    float y{0.0F};
    float z{0.0F};
};

int main() {
    Vector3 Origin;             // 0, 0, 0
    Vector3 Point{4.0F, 5.0F};  // 4, 5, 0
}

Default member initializer giúp object vẫn có trạng thái dễ đoán khi một số giá trị không được truyền vào.

Tóm lại

Struct là một type đầy đủ của C++. Dùng nó cho dữ liệu đơn giản; aggregate initialization điền giá trị theo đúng thứ tự member.

BÀI 05

Operator Overloading

Operator overloading cho phép những expression như A + B hoạt động với type do ta tự tạo.

1. Vấn đề: C++ chưa biết cộng hai Vector3

cpp
Vector3 Position{1.0F, 2.0F, 3.0F};
Vector3 Movement{4.0F, 5.0F, 6.0F};

Vector3 Result{Position + Movement}; // Chưa hợp lệ

Compiler biết cộng int và float, nhưng chưa biết dấu + phải làm gì với Vector3. Ta phải định nghĩa quy tắc đó.

2. Viết operator+ như một function

cpp
struct Vector3 {
    float x;
    float y;
    float z;
};

Vector3 operator+(const Vector3& Left, const Vector3& Right) {
    return {
        Left.x + Right.x,
        Left.y + Right.y,
        Left.z + Right.z
    };
}

Đọc signature này từng phần:

  • Vector3 đầu tiên: function trả về một object Vector3 mới.
  • operator+: đây là function triển khai toán tử +.
  • Left: operand nằm bên trái dấu +.
  • Right: operand nằm bên phải dấu +.
  • const Vector3&: không copy object và không cho function sửa object gốc.

Cú pháp operator

Position + Movement

Cách hiểu tương đương

operator+(
    Position,
    Movement
)

3. Chạy một ví dụ hoàn chỉnh

cpp
#include <iostream>

struct Vector3 {
    float x;
    float y;
    float z;
};

Vector3 operator+(const Vector3& Left, const Vector3& Right) {
    return {
        Left.x + Right.x,
        Left.y + Right.y,
        Left.z + Right.z
    };
}

int main() {
    Vector3 Position{1.0F, 2.0F, 3.0F};
    Vector3 Movement{4.0F, 5.0F, 6.0F};
    Vector3 Result{Position + Movement};

    std::cout << Result.x << ", "
              << Result.y << ", "
              << Result.z;
}

Output là 5, 7, 9. Mỗi member của object bên trái được cộng với member tương ứng của object bên phải.

4. Thứ tự operand có ý nghĩa

cpp
Vector3 operator*(const Vector3& Vector, float Scale);
Vector3 operator*(float Scale, const Vector3& Vector);
  • Overload đầu phục vụ Vector * 2.0F.
  • Overload thứ hai phục vụ 2.0F * Vector.
  • C++ không tự đảo operand, kể cả khi phép toán có vẻ giao hoán.

5. Viết operator dưới dạng member function

cpp
struct Vector3 {
    float x;
    float y;
    float z;

    Vector3 operator+(const Vector3& Other) const {
        return {
            x + Other.x,
            y + Other.y,
            z + Other.z
        };
    }
};

Expression

A + B

Cách hiểu

A.operator+(B)

Object A chính là operand bên trái, nên member function chỉ nhận B làm parameter. Từ khóa const cuối signature cam kết function không sửa A.

6. Unary operator

Unary operator chỉ có một operand. Khi viết dưới dạng member function, nó không cần parameter.

cpp
struct Vector3 {
    float x;
    float y;
    float z;

    Vector3 operator-() const {
        return {-x, -y, -z};
    }
};

Vector3 Direction{1.0F, -2.0F, 3.0F};
Vector3 Opposite{-Direction}; // -1, 2, -3

7. Những thứ operator overloading không làm được

  • Không tạo ra toán tử hoàn toàn mới.
  • Không đổi độ ưu tiên của toán tử.
  • Không đổi số lượng operand mà toán tử yêu cầu.
  • Ít nhất một operand phải là user-defined type.
  • Không nên gán ý nghĩa trái trực giác; + nên tạo kết quả kiểu “cộng”, không phải xóa dữ liệu.
Tóm lại

Operator overloading chỉ là function có tên đặc biệt. Expression A + B được chuyển thành lời gọi function mà ta định nghĩa.

BÀI 06

Structured Binding

Structured binding đặt tên riêng cho từng phần của một object mà không phải viết nhiều dòng truy cập member.

1. Từ ba dòng xuống một dòng

Truy cập thủ công

float x = Position.x;
float y = Position.y;
float z = Position.z;

Structured binding

auto [x, y, z]
    = Position;

Cả hai đoạn đều tạo ra ba tên x, y, z. Structured binding chỉ viết gọn hơn.

2. Ví dụ cơ bản — yêu cầu C++17

cpp
#include <iostream>

struct Vector3 {
    float x;
    float y;
    float z;
};

int main() {
    Vector3 Position{1.0F, 2.0F, 3.0F};
    auto [x, y, z] = Position;

    std::cout << x << ' '
              << y << ' '
              << z;
}
  • auto để compiler tự suy ra type của các biến mới.
  • [x, y, z] là danh sách tên mới.
  • = Position là object bị tách.
  • Output là 1 2 3.

3. Ghép theo vị trí, không theo tên

cpp
auto [First, Second, Third] = Position;
Firstnhận member đầu tiên: Position.x
Secondnhận member thứ hai: Position.y
Thirdnhận member thứ ba: Position.z

Tên biến mới không cần giống tên member. Điều quyết định là thứ tự khai báo member trong struct.

4. Copy, reference và const reference

cpp
Vector3 Position{1.0F, 2.0F, 3.0F};

// 1. Copy
{
    auto [x, y, z] = Position;
    x = 99.0F;
    // Position.x vẫn là 1.0F
}

// 2. Reference
{
    auto& [x, y, z] = Position;
    x = 99.0F;
    // Position.x trở thành 99.0F
}

// 3. Const reference
{
    const auto& [x, y, z] = Position;
    // Chỉ đọc, không copy, không được sửa x/y/z
}
Cú phápCó copy?Sửa object gốc?Khi dùng
auto [...]CóKhôngCần bản sao độc lập
auto& [...]KhôngCóMuốn sửa member gốc
const auto& [...]KhôngKhôngChỉ đọc object

5. Trường hợp thực tế với std::pair

cpp
#include <iostream>
#include <string>
#include <utility>

std::pair<std::string, int> GetPlayer() {
    return {"Chuong", 42};
}

int main() {
    auto [Name, Level] = GetPlayer();

    std::cout << Name << " - level " << Level;
}

Thay vì dùng Result.first và Result.second, ta đặt ngay hai tên có nghĩa là Name và Level.

6. Giới hạn cần nhớ

  • Với struct đơn giản, số biến thường phải khớp số member được tách.
  • Không tự động bỏ qua một member ở giữa.
  • Không unpack private member theo cách mặc định.
  • Structured binding dùng tốt với struct, array, std::pair, std::tuple và kết quả trả về nhiều giá trị.
Chốt chương

Structured binding tách object thành các tên dễ dùng. Chọn copy, reference hoặc const reference dựa trên việc có cần sửa object gốc hay không.
