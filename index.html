BÀI 01

Abstraction và Class

Abstraction giúp ta mô tả một nhóm đối tượng bằng một kiểu chung. Class là công cụ trung tâm để làm việc đó trong C++.

Abstraction

Khi nhiều đối tượng có chung dữ liệu và hành vi, ta không nên quản lý chúng bằng các biến và hàm rời rạc. Ta tạo một kiểu dữ liệu mô tả phần chung, rồi tạo các object cụ thể từ kiểu đó.

StateDữ liệu mô tả trạng thái hiện tại, chẳng hạn máu hoặc tên.
BehaviorHành vi object có thể thực hiện, chẳng hạn nhận sát thương.

Built-in Type và User-defined Type

C++ cung cấp các kiểu có sẵn như int, float và bool. Khi tự định nghĩa một class, ta tạo ra một user-defined type.

cpp
class Monster {
public:
    int Health{150};

    void TakeDamage(int Damage) {
        Health -= Damage;
    }
};

Class, Object và Instance

Monster là class. Bonker và Basher là object, hay instance, được tạo từ class đó. Quá trình tạo object từ class gọi là instantiation.

cpp
int main() {
    Monster Bonker;
    Monster Basher;
}

Class Member và toán tử .

Biến trong class là data member; hàm trong class là member function. Dùng toán tử . để truy cập member của một object.

cpp
#include <iostream>

int main() {
    Monster Bonker;
    std::cout << Bonker.Health << '\n';
    Bonker.TakeDamage(25);
    std::cout << Bonker.Health << '\n';
}

Mỗi object giữ bản sao riêng của data member. Thay đổi Bonker.Health không làm thay đổi Basher.Health.

Function Prototype trong Class

Member function có thể được khai báo trong class rồi định nghĩa bên ngoài bằng toán tử phân giải phạm vi ::.

cpp
class Monster {
public:
    void TakeDamage(int Damage);

private:
    int Health{150};
};

void Monster::TakeDamage(int Damage) {
    Health -= Damage;
}
Tóm lại

Class là bản mô tả chung; object là thực thể cụ thể. Class gom state và behavior vào cùng một kiểu.

BÀI 02

Encapsulation và Access Specifier

Encapsulation bảo vệ trạng thái hợp lệ của object và tạo ra một API rõ ràng.

Class Invariant và Contract

Class invariant là quy tắc mà object luôn phải giữ, ví dụ máu không thể nhỏ hơn 0. Contract là lời đảm bảo mà code cung cấp cho bên sử dụng.

cpp
void TakeDamage(int Damage) {
    Health -= Damage;
    if (Health < 0) {
        Health = 0;
    }
}

Public và Private Member

public cho phép code bên ngoài truy cập. private chỉ cho phép member function của chính class truy cập. Trong class, mức truy cập mặc định là private.

cpp
class Monster {
public:
    void TakeDamage(int Damage) {
        Health -= Damage;
        if (Health < 0) Health = 0;
    }

private:
    int Health{150};
};

Getter và Setter

Getter đọc dữ liệu private. Setter cập nhật dữ liệu thông qua quy tắc kiểm tra của class.

cpp
class Monster {
public:
    int GetHealth() const {
        return Health;
    }

    void SetHealth(int NewHealth) {
        Health = NewHealth < 0 ? 0 : NewHealth;
    }

private:
    int Health{150};
};
Không tạo getter/setter theo phản xạ.

Chỉ công khai thao tác mà code bên ngoài thực sự cần. Setter vô điều kiện cho mọi field sẽ làm mất ý nghĩa của private.

API của Class

Tập hợp public member tạo nên API của class. Chi tiết private có thể thay đổi mà không buộc code sử dụng class phải sửa theo.

Tóm lại

Encapsulation giữ invariant, giảm phụ thuộc và giới hạn những thao tác mà code bên ngoài được phép thực hiện.

BÀI 03

Constructor và Destructor

Constructor thiết lập trạng thái ban đầu; destructor chạy khi vòng đời object kết thúc.

Constructor

Constructor có cùng tên với class và không có return type. Nó thường nằm trong vùng public để code bên ngoài có thể tạo object.

cpp
#include <iostream>

class Monster {
public:
    Monster() {
        std::cout << "Ready for Battle!";
    }
};

int main() {
    Monster Goblin;
}

Constructor Parameter

cpp
#include <string>

class Monster {
public:
    Monster(std::string Name, int Health)
        : mName{Name}, mHealth{Health} {}

private:
    std::string mName;
    int mHealth;
};

Monster Goblin{"Bonker", 150};

Tiền tố m chỉ là naming convention để phân biệt member với parameter.

Nhiều Constructor và Default Argument

Một class có thể overload constructor nếu danh sách parameter khác nhau. Default argument thường giúp gộp các overload gần giống nhau.

cpp
Monster(std::string Name, int Health = 150)
    : mName{Name}, mHealth{Health} {}

Monster Bonker{"Bonker"};
Monster Basher{"Basher", 250};

Ambiguous Constructor Call

Tên parameter không tham gia vào overload resolution. Hai constructor có cùng kiểu parameter là trùng chữ ký.

cpp
class Monster {
public:
    Monster(int Level) {}
    Monster(int Health) {} // Lỗi: cùng chữ ký Monster(int)
};

Default Constructor

Nếu class không khai báo constructor, compiler có thể sinh default constructor. Sau khi khai báo custom constructor, có thể yêu cầu tạo lại bằng = default.

cpp
class Monster {
public:
    Monster() = default;
    Monster(std::string Name, int Health = 150)
        : mName{Name}, mHealth{Health} {}

private:
    std::string mName;
    int mHealth{150};
};

Destructor

cpp
#include <iostream>

class Monster {
public:
    Monster() { std::cout << "Monster Created\n"; }
    ~Monster() { std::cout << "Monster Destroyed\n"; }
};

void SomeFunction() {
    Monster Goblin;
} // Goblin bị hủy tại đây
Tóm lại

Constructor chạy khi object được tạo; destructor chạy khi object bị hủy. Mỗi lời gọi constructor phải khớp duy nhất một overload.

BÀI 04

Struct và Aggregate Initialization

Struct phù hợp với những kiểu dữ liệu đơn giản, chủ yếu dùng để gom các giá trị liên quan.

Tạo Struct

cpp
struct Vector3 {
    float x;
    float y;
    float z;
};

Vector3 Position;

Struct và Class: khác biệt kỹ thuật

StructClass
Member mặc địnhpublicprivate
Inheritance mặc địnhpublicprivate

Ngoài hai mặc định này, struct và class có gần như cùng khả năng ngôn ngữ.

Khác biệt về ý nghĩa sử dụng

Thông lệ là dùng struct cho dữ liệu đơn giản với member public; dùng class khi cần invariant, encapsulation và hành vi phức tạp.

Aggregate Initialization

Aggregate initialization cho phép cung cấp trực tiếp giá trị member.

cpp
Vector3 Position{1.0F, 2.0F, 3.0F};

// x = 1.0F, y = 2.0F, z = 3.0F

Giá trị được ghép theo thứ tự member, không theo tên biến.

Tóm lại

Struct thường đại diện cho một nhóm dữ liệu đơn giản. Aggregate initialization giúp khởi tạo nhanh các member public theo thứ tự.

BÀI 05

Operator Overloading

Operator overloading làm cho object có thể sử dụng cú pháp tự nhiên như +, - hoặc ==.

Operator là Function

Biểu thức dùng toán tử có thể được hiểu như lời gọi một function đặc biệt.

cpp
struct Vector3 {
    float x;
    float y;
    float z;
};

Vector3 operator+(const Vector3& Left, const Vector3& Right) {
    return {
        Left.x + Right.x,
        Left.y + Right.y,
        Left.z + Right.z
    };
}

Thứ tự Operand

Với operator nhị phân, operand bên trái và bên phải được truyền theo đúng thứ tự. Không phải phép toán nào cũng giao hoán.

cpp
Vector3 Result = Position + Offset;
// Tương đương operator+(Position, Offset)

Overload bằng Member Function

Khi operator là member function, object bên trái đóng vai trò *this; chỉ operand bên phải xuất hiện trong danh sách parameter.

cpp
struct Vector3 {
    float x;
    float y;
    float z;

    Vector3 operator+(const Vector3& Other) const {
        return {x + Other.x, y + Other.y, z + Other.z};
    }
};

Unary Operator

cpp
Vector3 operator-() const {
    return {-x, -y, -z};
}

Prototype, Definition và Chaining

Operator có thể được khai báo trong type rồi định nghĩa bên ngoài. Để chaining hoạt động tự nhiên, return type phải phù hợp với operator tiếp theo.

cpp
Vector3 A{1, 2, 3};
Vector3 B{4, 5, 6};
Vector3 C{7, 8, 9};
Vector3 Result = A + B + C;
Tóm lại

Operator overloading nên tạo ra hành vi trực quan, nhất quán với ý nghĩa quen thuộc của toán tử.

BÀI 06

Structured Binding

Structured binding giúp unpack struct, array, std::pair và std::tuple.

C++ Language Standard

Structured binding được thêm trong C++17. Dự án phải biên dịch với standard phù hợp.

Sử dụng Structured Binding

cpp
Vector3 Position{1.0F, 2.0F, 3.0F};
auto [x, y, z] = Position;

Các biến được ghép theo vị trí khai báo member, không theo tên.

Copy, Reference và Const Reference

cpp
auto [x1, y1, z1] = Position;          // copy
auto& [x2, y2, z2] = Position;         // reference
const auto& [x3, y3, z3] = Position;   // const reference
Cú phápCopy?Sửa object gốc?
auto [...]CóKhông
auto& [...]KhôngCó
const auto& [...]KhôngKhông

Giới hạn

  • Số biến thường phải khớp số thành phần được tách.
  • Không tự động bỏ qua một member ở giữa.
  • Không unpack private member theo cơ chế mặc định.

Ví dụ với std::pair

cpp
#include <string>
#include <utility>

std::pair<std::string, int> GetPlayer() {
    return {"Chuong", 42};
}

int main() {
    auto [Name, Level] = GetPlayer();
}
Tóm lại

Structured binding tạo tên rõ nghĩa cho các thành phần. Chọn copy, reference hoặc const reference theo nhu cầu sử dụng.

BÀI 07

Inheritance

Inheritance mô hình hóa quan hệ “là một” giữa các type.

Vì sao cần Inheritance?

Khi nhiều class có chung member và behavior, việc sao chép code tạo ra trùng lặp. Inheritance cho phép đặt phần chung vào base class, rồi để các derived class kế thừa.

Không kế thừa

class Goblin {
public:
    void Render();
};

class Dragon {
public:
    void Render();
};

Có kế thừa

class Actor {
public:
    void Render();
};

class Goblin : public Actor {};
class Dragon : public Actor {};

Parent, Child, Base và Derived Class

Class được kế thừa thường gọi là parent hoặc base class. Class kế thừa gọi là child, subclass hoặc derived class.

Implement Inheritance

cpp
class Actor {
public:
    void Render() const {}
};

class Goblin : public Actor {
public:
    void Attack() {}
};

int main() {
    Goblin Bonker;
    Bonker.Render(); // inherited member
    Bonker.Attack();
}

public ở đây là access mode của inheritance. Với public inheritance, public member của base class tiếp tục là public trong derived class.

Inheritance Relationship

Chỉ dùng inheritance khi derived type thực sự “là một” base type. Goblin là một Actor hợp lý; Weapon là một Actor có thể không hợp lý tùy thiết kế.

Nhiều tầng Inheritance

cpp
class Actor {};
class Character : public Actor {};
class Monster : public Character {};
class Goblin : public Monster {};

Goblin nhận các member có thể kế thừa từ toàn bộ chuỗi base class.

Ngăn Inheritance bằng final

cpp
class Goblin final : public Monster {};

// class EliteGoblin : public Goblin {}; // Lỗi
Tóm lại

Inheritance tái sử dụng phần chung và thể hiện quan hệ “là một”. Tránh dùng nó chỉ để né viết vài dòng code.

BÀI 08

Protected Member

Protected member nằm giữa public và private.

Giới hạn truy cập khi Inheritance

Derived class không thể truy cập trực tiếp private member của base class. Điều này vẫn đúng dù object derived có chứa phần base bên trong.

cpp
class Actor {
private:
    int Health{100};
};

class Goblin : public Actor {
public:
    void TakeDamage(int Damage) {
        // Health -= Damage; // Lỗi: Health là private
    }
};

Dùng protected

cpp
class Actor {
protected:
    int Health{100};
};

class Goblin : public Actor {
public:
    void TakeDamage(int Damage) {
        Health -= Damage; // Hợp lệ trong derived class
    }
};

int main() {
    Goblin Bonker;
    // Bonker.Health = 500; // Lỗi từ bên ngoài
}

Chọn Access Specifier

SpecifierClassDerived classBên ngoài
publicCóCóCó
protectedCóCóKhông
privateCóKhôngKhông
Ưu tiên private khi có thể.

protected làm derived class phụ thuộc vào chi tiết triển khai của base class. Một public/protected function có kiểm soát thường an toàn hơn việc mở trực tiếp dữ liệu.

Tóm lại

Protected member cho derived class quyền truy cập nhưng vẫn chặn code bên ngoài. Dùng có chủ đích để không phá encapsulation.

BÀI 09

Member Initializer List

Member initializer list là cách chuẩn để khởi tạo member trong constructor.

Định nghĩa Member Initializer List

cpp
class Monster {
public:
    Monster(std::string Name, int Health)
        : mName{Name}, mHealth{Health} {
    }

private:
    std::string mName;
    int mHealth;
};

Danh sách nằm sau dấu : và trước constructor body. Mỗi member được khởi tạo bằng biểu thức trong dấu ngoặc nhọn.

Vì sao nên dùng?

PerformanceKhởi tạo trực tiếp thay vì default-initialize rồi gán lại.
ReadabilityCho thấy ngay trạng thái ban đầu của object.
RequiredBắt buộc với reference, const member và base class không có default constructor.

Khởi tạo nhiều Member

cpp
Monster(std::string Name, int Health, int Level)
    : mName{Name},
      mHealth{Health},
      mLevel{Level} {
}
Thứ tự thực tế theo khai báo member.

Dù viết initializer theo thứ tự nào, C++ vẫn khởi tạo member theo thứ tự chúng được khai báo trong class.

Dùng Expression và Parameter

cpp
Monster(std::string Name, int Level)
    : mName{std::move(Name)},
      mLevel{Level},
      mHealth{Level * 50} {
}

Initializer có thể dùng parameter và expression, miễn biểu thức tạo ra kiểu phù hợp cho member.

Trường hợp bắt buộc

cpp
class Example {
public:
    Example(int Value, int& Target)
        : Constant{Value}, Reference{Target} {}

private:
    const int Constant;
    int& Reference;
};

const member và reference phải được khởi tạo, không thể để trống rồi gán trong constructor body.

Tóm lại

Member initializer list rõ ràng, hiệu quả và đôi khi bắt buộc. Hãy ưu tiên nó thay cho gán trong constructor body.

BÀI 10

Làm việc với Inherited Member

Derived class có thể kế thừa constructor, gọi constructor của base class, cập nhật inherited variable và mở rộng inherited function.

Kế thừa Constructor

Constructor không tự động được kế thừa theo cách member function thông thường được kế thừa. Có thể đưa constructor của base class vào derived class bằng using.

cpp
class Actor {
public:
    Actor(int Health) : mHealth{Health} {}

protected:
    int mHealth;
};

class Goblin : public Actor {
public:
    using Actor::Actor;
};

Goblin Bonker{150};

Gọi Constructor của Base Class

Khi derived class có constructor riêng, constructor của base class được gọi trong member initializer list.

cpp
class Goblin : public Actor {
public:
    Goblin(std::string Name, int Health)
        : Actor{Health}, mName{std::move(Name)} {}

private:
    std::string mName;
};

Base class luôn được khởi tạo trước member của derived class và trước constructor body của derived class.

Constructor Call Order

output
Actor constructor
Goblin constructor
Goblin destructor
Actor destructor

Khi tạo object: base constructor chạy trước, derived constructor chạy sau. Khi hủy object, thứ tự đảo ngược.

Cập nhật Inherited Variable

cpp
class Actor {
protected:
    int Health{100};
};

class Goblin : public Actor {
public:
    void TakeDamage(int Damage) {
        Health -= Damage;
    }
};

Lỗi thường gặp: Shadowing Variable

Nếu derived class khai báo member cùng tên với base class, member mới che khuất member cũ thay vì thay thế nó.

cpp
class Actor {
protected:
    int Health{100};
};

class Goblin : public Actor {
private:
    int Health{250}; // member khác, che khuất Actor::Health
};

Có thể chỉ rõ member base bằng Actor::Health, nhưng thiết kế này dễ gây nhầm và thường nên tránh.

Shadowing Inherited Function

Khi derived class khai báo function cùng tên, các overload cùng tên của base class có thể bị che khuất.

cpp
class Actor {
public:
    void Render() const {}
    void Render(int DetailLevel) const {}
};

class Goblin : public Actor {
public:
    using Actor::Render;
    void Render() const {}
};

using Actor::Render đưa các overload của base class trở lại overload set trong derived class.

Mở rộng Inherited Function

Derived function có thể gọi phiên bản của base class rồi thêm hành vi riêng bằng tên có phạm vi.

cpp
class Actor {
public:
    void Render() const {
        // Vẽ phần chung
    }
};

class Goblin : public Actor {
public:
    void Render() const {
        Actor::Render();
        // Vẽ thêm chi tiết riêng của Goblin
    }
};

Trong Unreal Engine, macro hoặc quy ước có thể cung cấp alias Super, nhưng C++ chuẩn dùng trực tiếp tên base class như Actor::Render().

Tóm lại

Derived class có thể tái sử dụng và mở rộng inherited member, nhưng phải hiểu rõ constructor order, shadowing và cách gọi rõ ràng phiên bản của base class.
