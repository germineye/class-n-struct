BÀI 01

Abstraction và Class

Thay vì quản lý từng đối tượng bằng một đống biến và hàm rời rạc, ta định nghĩa một kiểu dữ liệu chung rồi tạo ra bao nhiêu đối tượng tùy ý.

1. Vì sao cần abstraction?

Giả sử game có 100 con quái. Mỗi con đều có máu và có thể nhận sát thương. Viết riêng 100 biến và 100 hàm là cách tự hành xác. Ta gom đặc điểm chung đó thành một kiểu Monster.

StateTrạng thái hiện tại của object, ví dụ Health.
BehaviorHành vi object có thể thực hiện, ví dụ TakeDamage().

User-defined type là kiểu dữ liệu do chính ta tạo để mô tả đối tượng trong chương trình.

2. Khai báo class và tạo object

cpp
class Monster {
public:
    int Health{150};

    void TakeDamage(int Damage) {
        Health -= Damage;
    }
};

int main() {
    Monster Bonker;
    Monster Basher;
}
  • Monster là class.
  • Bonker và Basher là object, đồng thời cũng được gọi là instance.
  • Quá trình tạo object từ class gọi là instantiation.
  • Dấu chấm phẩy sau dấu } của class là bắt buộc.

3. Class member và toán tử .

Biến và hàm nằm bên trong class được gọi chung là class member. Biến có thể được gọi là data member, field hoặc property; hàm thường được gọi là member function hoặc method.

cpp
#include <iostream>

int main() {
    Monster Bonker;

    std::cout << Bonker.Health << '\n';
    Bonker.TakeDamage(25);
    std::cout << Bonker.Health << '\n';
}

Toán tử . là member access operator. Mỗi object có bản sao riêng của các data member, nên thay đổi Bonker.Health không ảnh hưởng đến Basher.Health.

4. Tách phần khai báo và định nghĩa hàm

Member function có thể được khai báo trong class rồi định nghĩa bên ngoài bằng scope resolution operator ::.

cpp
class Monster {
public:
    void TakeDamage(int Damage);

private:
    int Health{150};
};

void Monster::TakeDamage(int Damage) {
    Health -= Damage;
}
Tóm lại

Class là bản mô tả chung; object là thực thể cụ thể. Class gom state và behavior vào cùng một kiểu dữ liệu.

BÀI 02

Encapsulation và Access Specifier

Đừng để code bên ngoài chọc thẳng vào dữ liệu nội bộ. Hãy buộc mọi thay đổi đi qua một lối vào có kiểm soát.

1. Class invariant và contract

Class invariant là một quy tắc mà object luôn phải giữ, chẳng hạn: Health không bao giờ âm. Một lời đảm bảo như vậy cũng có thể được gọi là contract.

cpp
void TakeDamage(int Damage) {
    Health -= Damage;

    if (Health < 0) {
        Health = 0;
    }
}

Hàm trên giữ được invariant, nhưng vô nghĩa nếu bên ngoài vẫn có thể viết Goblin.Health = -999;. Vì vậy dữ liệu cần được ẩn.

2. public và private

cpp
class Monster {
public:
    void TakeDamage(int Damage) {
        Health -= Damage;

        if (Health < 0) {
            Health = 0;
        }
    }

private:
    int Health{150};
};
  • public: code bên ngoài object được phép truy cập.
  • private: chỉ member function của chính class được truy cập.
  • Trong class, member mặc định là private.
  • Một class có thể có nhiều vùng public và private, nhưng chia quá vụn thường làm code khó đọc.

3. Getter và setter

Getter cho phép đọc dữ liệu private. Setter cho phép thay đổi dữ liệu nhưng vẫn áp dụng kiểm tra.

cpp
class Monster {
public:
    int GetHealth() const {
        return Health;
    }

    void SetHealth(int NewHealth) {
        Health = NewHealth < 0 ? 0 : NewHealth;
    }

private:
    int Health{150};
};
Đừng tạo getter/setter theo phản xạ.

Chỉ công khai thao tác mà code bên ngoài thực sự cần. Nếu mọi field đều có setter vô điều kiện thì private gần như mất tác dụng.

4. API của class

API của class gồm tên class và các public member. Phần private là implementation detail: có thể thay đổi mà không buộc code sử dụng class phải sửa theo.

Tóm lại

Dữ liệu private, hành vi public. Class tự chịu trách nhiệm giữ object ở trạng thái hợp lệ.

BÀI 03

Constructor và Destructor

Object nên hợp lệ ngay từ lúc sinh ra. Constructor nhận dữ liệu ban đầu và thiết lập toàn bộ member trước khi object được sử dụng.

1. Constructor cơ bản

Constructor có cùng tên với class và không có return type, kể cả void. Constructor thường phải nằm trong vùng public để code bên ngoài tạo được object.

cpp
#include <string>

class Monster {
public:
    Monster(std::string Name, int Health)
        : Name{Name}, Health{Health} {}

private:
    std::string Name;
    int Health;
};

int main() {
    Monster Goblin{"Bonker", 150};
}

2. Member initializer list

Phần sau dấu : gọi là member initializer list. Đây là cách chuẩn để khởi tạo data member.

Khởi tạo đúng nghĩa

Monster(std::string Name)
    : Name{Name} {}

Gán sau khi đã khởi tạo

Monster(std::string Name) {
    this->Name = Name;
}

Initializer list là bắt buộc với một số loại member như reference, const member hoặc object không có default constructor.

3. Nhiều constructor và default argument

cpp
class Monster {
public:
    Monster() = default;

    Monster(std::string Name, int Health = 150)
        : Name{Name}, Health{Health} {}

private:
    std::string Name{"Unknown"};
    int Health{150};
};
  • Default constructor là constructor dùng được với lời gọi không có argument.
  • Khi tự khai báo constructor có tham số, compiler thường không còn tự sinh default constructor. Dùng = default nếu vẫn cần nó.
  • Có thể overload constructor bằng các danh sách parameter khác nhau.
  • Không được để hai constructor cùng khớp một lời gọi; compiler sẽ báo ambiguity.

4. Destructor

Destructor có tên class kèm dấu ~, không có return type và không nhận parameter. Nó chạy khi object rời khỏi phạm vi sống hoặc bị hủy.

cpp
#include <iostream>

class Session {
public:
    Session() {
        std::cout << "Session bat dau\n";
    }

    ~Session() {
        std::cout << "Session ket thuc\n";
    }
};

Với class chỉ chứa std::string, std::vector và các kiểu quản lý tài nguyên tự động, thường không cần tự viết destructor. Destructor quan trọng khi class trực tiếp sở hữu tài nguyên cần dọn dẹp.

Tóm lại

Constructor tạo trạng thái hợp lệ; destructor dọn tài nguyên. Ưu tiên member initializer list và tránh overload nhập nhằng.

BÀI 04

Struct và Aggregate Initialization

Struct hợp với những kiểu dữ liệu đơn giản chủ yếu dùng để chứa và truyền dữ liệu.

1. Tạo struct

cpp
struct Vector3 {
    float x;
    float y;
    float z;
};

int main() {
    Vector3 Position;
    Position.x = 1.0F;
}

Về kỹ thuật, struct và class gần như giống hệt nhau. Struct vẫn có member function, constructor và destructor.

2. Khác biệt giữa struct và class

structclass
Quyền truy cập mặc địnhpublicprivate
Cách dùng phổ biếnData đơn giản, ít invariantKiểu có hành vi và quy tắc nội bộ
Khả năng kỹ thuậtGần như tương đương

Quy ước thực tế: dùng struct cho object “thụ động” chủ yếu mang dữ liệu; dùng class khi cần encapsulation, invariant hoặc hành vi phức tạp.

3. Aggregate initialization

cpp
struct Vector3 {
    float x;
    float y;
    float z;
};

int main() {
    Vector3 Position{1.9F, 2.6F, 0.3F};
}

Các giá trị được gán theo đúng thứ tự member: x, rồi y, rồi z. Cách này phù hợp với kiểu đơn giản. Khi type có private member hoặc thiết kế phức tạp, hãy dùng constructor thay vì cố biến nó thành aggregate.

Tóm lại

Struct không phải “class phiên bản yếu”. Khác biệt kỹ thuật chính là quyền truy cập mặc định; khác biệt lớn hơn nằm ở mục đích sử dụng.

BÀI 05

Operator Overloading

Cho phép object tự định nghĩa ý nghĩa của +, -, *… để code tự nhiên hơn, miễn là hành vi vẫn đúng với trực giác.

1. Operator thực chất là function

cpp
struct Vector3 {
    float x;
    float y;
    float z;
};

Vector3 operator+(const Vector3& Left, const Vector3& Right) {
    return {
        Left.x + Right.x,
        Left.y + Right.y,
        Left.z + Right.z
    };
}

int main() {
    Vector3 Position{1.0F, 2.0F, 3.0F};
    Vector3 Movement{4.0F, 5.0F, 6.0F};
    Vector3 Result{Position + Movement};
}

Cú pháp Position + Movement được compiler chuyển thành lời gọi hàm operator+(Position, Movement). Truyền bằng const& tránh copy và đảm bảo function không sửa operand.

2. Thứ tự operand có ý nghĩa

Với free function, parameter đầu là operand bên trái, parameter thứ hai là operand bên phải.

cpp
Vector3 operator*(const Vector3& Vector, float Scale);
Vector3 operator*(float Scale, const Vector3& Vector);

Hai overload trên phục vụ hai biểu thức khác nhau: Vector * 2.0F và 2.0F * Vector. C++ không tự suy ra phép toán có tính giao hoán.

3. Operator dưới dạng member function

cpp
struct Vector3 {
    float x;
    float y;
    float z;

    Vector3 operator+(const Vector3& Other) const {
        return {
            x + Other.x,
            y + Other.y,
            z + Other.z
        };
    }
};

Member function chỉ nhận operand bên phải làm parameter; object gọi hàm chính là operand bên trái. Biểu thức A + B tương đương A.operator+(B).

4. Unary operator

Unary operator chỉ có một operand. Khi viết dưới dạng member function, nó không cần parameter.

cpp
struct Vector3 {
    float x;
    float y;
    float z;

    Vector3 operator-() const {
        return {-x, -y, -z};
    }
};

5. Nguyên tắc thiết kế

  • + thường trả về object mới, không âm thầm sửa operand.
  • += thường sửa operand bên trái và trả reference để hỗ trợ chaining.
  • Không làm + mang nghĩa “xóa dữ liệu” hoặc hành vi trái trực giác.
  • Chỉ overload khi nó làm code dễ đọc hơn một function có tên rõ ràng.
Tóm lại

Operator overloading là function có cú pháp đặc biệt. Giữ semantics quen thuộc quan trọng hơn việc “làm cho ngầu”.

BÀI 06

Structured Binding

Tách member của một object đơn giản thành nhiều biến chỉ bằng một dòng code.

1. Cú pháp cơ bản — yêu cầu C++17

cpp
#include <iostream>

struct Vector3 {
    float x;
    float y;
    float z;
};

int main() {
    Vector3 Position{1.0F, 2.0F, 3.0F};
    auto [x, y, z] = Position;

    std::cout << x << ' ' << y << ' ' << z;
}

auto là bắt buộc để compiler tự suy ra type của từng biến. Tên biến mới không cần giống tên member trong struct.

2. Ghép theo vị trí, không theo tên

cpp
auto [First, Second, Third] = Position;

First nhận member thứ nhất, Second nhận member thứ hai và Third nhận member thứ ba. Tên chỉ là tên mới do ta chọn.

3. Copy hay reference?

Tạo bản sao

auto [x, y, z] = Position;
x = 99.0F; // Position.x không đổi

Bind bằng reference

auto& [x, y, z] = Position;
x = 99.0F; // Position.x đổi theo

Dùng const auto& khi chỉ cần đọc mà không muốn copy object.

4. Giới hạn

  • Với kiểu aggregate đơn giản, thường phải khai báo đủ biến tương ứng với số member.
  • Không thể trực tiếp unpack các private member theo cách mặc định.
  • Với class phức tạp, member function có tên rõ ràng thường dễ kiểm soát hơn.
  • Structured binding hữu ích với struct, array, std::pair, tuple và kết quả trả về nhiều giá trị.
Chốt chương

Structured binding giúp unpack dữ liệu gọn hơn. Hãy chọn copy, reference hoặc const reference theo đúng mục đích.
